# Mass Uploader — Массовый залив кампаний TikTok

Заливаешь рекламные кампании в один или несколько кабинетов по сетапу. Следуй шагам строго по порядку.

---

## Шаг 0: Инициализация

1. Прочитай `config/accounts.md` — список кабинетов
2. Прочитай `config/safety_rules.md`
3. Прочитай `config/naming_rules.md`
4. Проверь историю за последние 3 дня в `config/history/`

---

## Шаг 1: Выбор сетапов и кабинетов

Показать пользователю список файлов из `config/setups/` (только `.md` без `_template`).

Запросить:
```
1. Какие сетапы заливаем? (можно выбрать несколько, через запятую) [список]
2. В какие кабинеты?
   A) Все активные кабинеты
   B) Выбрать конкретные
```

Прочитать все выбранные файлы сетапов. Запомнить их порядок — это порядок раундов залива.

---

## Шаг 2: Сбор обязательных параметров (запросить у пользователя)

Задать вопросы по очереди:

```
Перед заливом нужна информация:

1. Нейминг кампании (пример: TT_MYAPP_US_20250501)?
   →

2. Суб-параметр и базовое значение (пример: sub7=avfrites)?
   → Адсеты получат имена: avfrites1, avfrites2, avfrites3, avfrites4, avfrites5

3. Гео — в каких странах показываем? (одна или несколько, пример: US, CA, GB)?
   →

4. Бюджет на каждый адсет в день ($)?
   →

5. Приложение для этой кампании (название + платформа, если не указано в сетапе)?
   →

6. Время старта — стандарт 6:01 МСК следующих суток. Изменить?
   →
```

Сохранить ответы. Вычислить время старта: 6:01 МСК = 03:01 UTC. Перевести в формат кабинета (timezone из `accounts.md`).

## Шаг 3: Подготовка и проверка креативов

### 3.1 — Переименование файлов в папке

Получить список файлов из папки сетапа (`creatives/[название]/`).

Если папка пустая → остановиться:
```
❌ Папка creatives/[название]/ пустая. Добавь видео-файлы и повтори залив.
```

**Переименовать каждый файл** перед загрузкой чтобы имена не пересекались с предыдущими заливами:
- Формат: `[campaign_name]_V{INDEX}.[ext]`
- Пример: `TT_MYAPP_20250501_V1.mp4`, `TT_MYAPP_20250501_V2.mp4`

### 3.2 — Проверка библиотеки кабинета

Для каждого кабинета вызвать `tt_list_creative_assets` (asset_type: VIDEO).

Сравнить переименованные файлы с тем что уже есть в библиотеке по имени:
- Видео **уже есть** → запомнить его `video_id`, не загружать повторно
- Видео **отсутствует** → нужна загрузка

### 3.3 — Загрузка новых видео

MCP сервер не имеет инструмента загрузки видео. Загрузка выполняется через Bash напрямую в TikTok API.

**Основной способ — curl через Bash:**

Для каждого видео выполнить:
```bash
curl -X POST "https://business-api.tiktok.com/open_api/v1.3/file/video/ad/upload/" \
  -H "Access-Token: [TOKEN из settings.json]" \
  -F "advertiser_id=[ID кабинета]" \
  -F "file_name=[переименованное имя файла]" \
  -F "video_file=@[полный путь к файлу]"
```

Из ответа извлечь `data.video_id` — сохранить в карту.

Если загрузка одного файла прошла успешно — продолжить с остальными. При ошибке — вывести сообщение и спросить пользователя как продолжить.

**Запасной способ (только если curl недоступен или токен не работает):**
```
⚠️  Не удалось загрузить видео через API. Загрузи вручную:
TikTok Ads Manager → Библиотека → Видео → Загрузить
Файлы: [список]
После загрузки пришли video_id для каждого файла.
```

### 3.4 — Итог: карта video_id

После завершения шага 3 должна быть готова карта:
```
Файл                        | video_id (кабинет ACC_001) | video_id (кабинет ACC_002)
TT_MYAPP_20250501_V1.mp4   | v09044g40000XXXXXXXXXX     | v09044g40000YYYYYYYYYY
TT_MYAPP_20250501_V2.mp4   | v09044g40000AAAAAAAAAA     | v09044g40000BBBBBBBBBB
```

## Шаг 5: Создание кампании

> **Темп работы:** 5 секунд между каждым API-вызовом. 15 секунд между кабинетами. Соблюдать строго.

> **Порядок залива — round-robin:**
> ```
> для каждого раунда (сетапа) 1..N:
>   для каждого кабинета 1..M:
>     создать кампанию + адсеты + объявления
>     ждать 15 сек (если есть следующий кабинет в раунде)
>   ждать 15 сек (если есть следующий раунд)
> ```
> Не заливать все раунды в один кабинет подряд — только так, как описано выше.

**Текущий раунд и кабинет:**

### 5.1 — Параметры кампании

```json
tt_create_campaign:
  advertiser_id: [из кабинета]
  campaign_name: [нейминг из шага 2]
  objective_type: APP_PROMOTION        // Продвижение приложения
  campaign_type: REGULAR_CAMPAIGN      // Ручная настройка (не Smart+)
  budget_optimize_on: false            // Бюджет на адсетах (ABO)

  // Только для iOS:
  ios14_quota_type: SLOT               // Включить специализированную кампанию iOS 14
  campaign_app_profile_page_state: OFF // Отключить страницу профиля приложения
  app_attribution_type: NON_SKAN       // Отключить атрибуцию SKAN
  // Для Android: эти три параметра НЕ передавать
```

**Проверка совместимости MCP:** Если `tt_create_campaign` вернул ошибку о неподдерживаемом `objective_type` — сообщить пользователю:
```
⚠️  MCP сервер не поддерживает APP_PROMOTION напрямую.
Варианты решения:
1. Создать кампанию вручную в TikTok Ads Manager, затем сообщить campaign_id
2. Обновить MCP сервер до версии с поддержкой APP_PROMOTION
```
Если пользователь предоставил `campaign_id` вручную — продолжить со шага 6.

Сохранить `campaign_id`.

## Шаг 6: Создание 5 адсетов

Создать ровно 5 адсетов. Каждый — копия предыдущего, отличается только именем.

### Имя адсета

Имя адсета всегда берётся из суб-параметра кампании — того самого значения что стоит после знака `=` в нейминге кампании, плюс порядковый номер в конце.

**Логика:**
1. Из нейминга кампании извлечь суб-значение (то что после `sub7=` или `sub_id_3=`)
2. Добавить к нему порядковый номер 1..5
3. Это и есть имя адсета

**Примеры:**
- Кампания содержит `sub7=avfrites` → адсеты: `avfrites1`, `avfrites2`, `avfrites3`, `avfrites4`, `avfrites5`
- Кампания содержит `sub_id_3=rocketavtoza` → адсеты: `rocketavtoza1`, `rocketavtoza2`, `rocketavtoza3`, `rocketavtoza4`, `rocketavtoza5`

Суб-значение в трекинговой ссылке каждого адсета = имя адсета (т.е. `avfrites1`, `avfrites2`, и т.д.)

### Параметры адсета

```json
tt_create_ad_group:
  advertiser_id: [из кабинета]
  campaign_id: [из шага 5]
  adgroup_name: "[sub_base][N]"        // avfrites1, avfrites2, ...

  // Приложение:
  // Android:
  promotion_type: APP_ANDROID
  app_id: [bundle_id из сетапа]
  // iOS:
  promotion_type: APP_IOS
  // (app_id НЕ передавать для iOS — поле отсутствует)

  // Размещение — ТОЛЬКО TikTok:
  placement_type: PLACEMENT_TYPE_NORMAL
  placements: ["PLACEMENT_TIKTOK"]
  // Если PLACEMENT_TIKTOK недоступен → остановить этот кабинет, сообщить пользователю:
  // "⚠️ Кабинет [NICKNAME]: TikTok placement недоступен. Пропускаем."

  // Гео:
  location_ids: [ID стран из шага 2]   // Получить ID через tt_search_locations

  // Возраст и пол (стандарт):
  age_groups: ["AGE_18_24", "AGE_25_34", "AGE_35_44", "AGE_45_54", "AGE_55_100"]
  gender: []                            // Оба пола (пустой = все)

  // Оптимизация — ВСЕГДА на покупку (стандарт):
  optimization_goal: IN_APP_EVENT
  conversion_event_code: PURCHASE
  // Если PURCHASE недоступен — спросить пользователя:
  // "⚠️ Кабинет [NICKNAME]: событие PURCHASE отсутствует. Варианты:
  //   1. REGISTRATION — регистрация
  //   2. INSTALL — установка
  //   3. Пропустить кабинет"

  // Ставка — автоматическая:
  bid_type: BID_TYPE_NO_BID

  // Бюджет:
  budget_mode: BUDGET_MODE_DAY
  budget: [сумма из шага 2]

  // Расписание:
  schedule_type: SCHEDULE_START_END
  schedule_start_time: "[дата следующего дня] 03:01:00"  // 6:01 МСК = 03:01 UTC

  // Трекинговая ссылка для APP кампаний НЕ передаётся.
  // Трекинг работает автоматически через AppsFlyer / Adjust:
  //   - Приложение подключено в Event Manager кабинета
  //   - Нейминг кампании содержит суб-параметр (sub7=avfrites) — по нему
  //     партнёрская сеть атрибутирует конверсии на своей стороне
  // click_tracking_url — только для WEBSITE сетапов
```

Повторить для всех 5 адсетов (N = 1..5). Сохранить все `adgroup_id[]`.

## Шаг 7: Создание объявлений

Для каждого из 5 адсетов создать **одно объявление** со всеми видео в `creative_list`.

При включённых "Умных креативах" (`SMART_CREATIVE`) TikTok сам комбинирует видео, тексты и CTA — поэтому все видео передаются в одном вызове `tt_create_ad`, а не отдельными объявлениями.

### 7.1 — Выбор персонального профиля

Перед созданием объявлений нужно определить `identity_id` — профиль TikTok, от имени которого будет показываться реклама.

**Шаг 1:** Спросить пользователя:
```
Используем профиль для приложения [app_name].

Есть ли уже профиль TikTok с именем "[app_name]" в кабинете [NICKNAME]?
TikTok Ads Manager → Инструменты → Профили TikTok

1. Да — пришли его ID
2. Нет — нужно создать новый
```

**Шаг 2 — если профиль есть:** принять ID, сохранить как `identity_id`.

**Шаг 3 — если профиля нет:** дать инструкцию по созданию:
```
Создай новый профиль:
TikTok Ads Manager → Инструменты → Профили TikTok → Добавить профиль

Название профиля: [app_name]  ← точно такое же как имя приложения
После создания пришли его ID.
```
Дождаться ID от пользователя, сохранить как `identity_id`.

**Важно:** один и тот же `identity_id` используется для всех 5 адсетов в этом кабинете.

---

### 7.2 — Параметры объявления

```json
tt_create_ad:
  advertiser_id: [из кабинета]
  adgroup_id: [adgroup_id текущего адсета]

  // Умные креативы — включить:
  creative_material_mode: SMART_CREATIVE

  // Spark Ads — выключить (CUSTOMIZED_USER, не TT_USER):
  identity_type: CUSTOMIZED_USER
  identity_id: [из шага 7.1]

  // Все видео в одном creative_list — Smart Creative сам тестирует комбинации:
  creative_list:
    - ad_name: "[campaign_name]_V1"
      ad_format: SINGLE_VIDEO
      video_id: [video_id V1 из карты шага 3]
      ad_text: "Play Now"
      call_to_action_id: DOWNLOAD

    - ad_name: "[campaign_name]_V2"
      ad_format: SINGLE_VIDEO
      video_id: [video_id V2 из карты шага 3]
      ad_text: "Start Play"
      call_to_action_id: INSTALL_NOW

    // ... добавить все видео из карты, чередуя тексты "Play Now" / "Start Play"
    // CTA выбирать по платформе:
    //   Android: DOWNLOAD, INSTALL_NOW, GOOGLE_PLAY
    //   iOS:     DOWNLOAD, INSTALL_NOW, APP_STORE
```

После завершения всех объявлений текущего кабинета — **ждать 15 секунд** перед переходом к следующему кабинету.

## Обработка ошибок

| Ошибка | Действие |
|--------|---------|
| TikTok placement недоступен | Пропустить кабинет, сообщить пользователю |
| PURCHASE event недоступен | Спросить альтернативу (REGISTRATION / INSTALL / пропустить) |
| Папка с креативами пустая | Остановить весь залив |
| Rate limit 429 | Ждать 60 сек, ретрай (макс 2 раза) |
| Ошибка API на кабинете | Залогировать, продолжить с остальными |
| Ошибка валидации | Остановить весь залив, показать ошибки |
