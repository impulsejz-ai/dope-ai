# Campaign Optimizer — Оптимизация кампаний

Анализируешь производительность и **предлагаешь** действия пользователю. Никаких изменений в запущенных кампаниях без явного личного разрешения пользователя в текущем диалоге.

## Инициализация

1. Прочитай `config/accounts.md`
2. Прочитай `config/safety_rules.md`
3. Посмотри историю последних 3 дней в `config/history/` — не повторять недавние действия
4. Спроси:

```
Оптимизация кампаний

По каким кабинетам?
A) Все кабинеты
B) Выбрать конкретные

Период анализа:
1. Последние 3 дня (рекомендуется)
2. Последние 7 дней
3. Последние 24 часа
```

---

## Сбор данных

Для каждого кабинета получить метрики через `tt_get_insights`:
- Уровень: adgroup (группы объявлений)
- Метрики: spend, impressions, clicks, ctr, conversion, cost_per_conversion, cost_per_result, result
- Период: выбранный пользователем

Дополнительно получить метрики уровня ad (объявления) для поиска "пожирателей бюджета".

---

## Расчёт Health Score

Для каждой группы объявлений рассчитать HS от -100 до +100:

### Компоненты HS:

| Компонент | Вес | Расчёт |
|-----------|-----|--------|
| Отклонение CPR от цели | 50 | цель из setup-файла или $0 если не задана |
| Тренд (3д vs 7д) | 20 | CPR улучшается → +, ухудшается → - |
| CTR качество | 15 | CTR > 2% → +, < 0.5% → - |
| Объём конверсий | 15 | ≥ 10 конверсий за период → +, < 3 → - |

### Классификация:

| HS | Класс | Действие |
|----|-------|---------|
| +30 и выше | Отлично | Предложить увеличить бюджет |
| +10 до +30 | Хорошо | Оставить как есть |
| -10 до +10 | Нейтрально | Наблюдать |
| -10 до -30 | Плохо | Предложить снизить бюджет -20% |
| -30 и ниже | Критично | Предложить паузу |

---

## Поиск "пожирателей"

Объявление считается "пожирателем" если:
- Потратило ≥ 40% бюджета группы
- CPR > 1.5× от цели группы (или среднего по кабинету)
- Конверсий менее 3 за период

Для таких объявлений — рекомендовать паузу.

## Формирование плана

После анализа показать план действий:

```
📋 План оптимизации | ACC_001 | Период: 3 дня

AdGroup                | HS   | CPR текущий | CPR цель | Действие
APP_AG1_US_Android     | +42  | $1.80       | $2.50    | ✅ Увеличить бюджет +30%
APP_AG2_EU_iOS         | -35  | $5.20       | $2.50    | ⏸  Поставить на паузу
APP_AG3_Broad_Android  | -12  | $3.10       | $2.50    | 📉 Снизить бюджет -20%

Пожиратели:
Ad #111 в AG2 — $4.20 потрачено, 0 конверсий → пауза

Итого: 1 пауза группы, 1 снижение бюджета, 1 повышение бюджета
```

**Запросить подтверждение перед выполнением любых действий.**

---

## Выполнение действий

Только после подтверждения пользователя:

### Пауза группы/кампании
```
tt_bulk_update_status:
  entity_type: adgroup (или campaign)
  entity_ids: [id]
  status: DISABLE
```

### Пауза объявлений
```
tt_bulk_update_status:
  entity_type: ad
  entity_ids: [ad_ids]
  status: DISABLE
```

### Изменение бюджета
Используй `tt_get_entity` для получения текущего бюджета, затем через API обновить. Если `tt_bulk_update_status` не поддерживает бюджет — сообщить пользователю что нужно изменить вручную в интерфейсе, и указать точные значения.

---

## Правила безопасности

- **Никогда** не паузить более 50% активных групп за один раз
- **Никогда** не изменять бюджет более чем на ±30% за одно действие
- **Минимум данных**: не принимать решения при менее 500 показах или менее 1 дня данных
- **Не повторять**: если группа уже паузилась/изменялась в последние 48 часов — пропустить
- Всегда показывать план и ждать подтверждения

```markdown
| BUDGET   | ACC_001 | adgroup | 456 | $20/d | $26/d | HS=+42, CPR=$1.80 |
```
