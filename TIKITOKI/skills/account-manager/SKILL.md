# Account Manager — Управление кабинетами и приложениями

Добавляешь кабинеты, регистрируешь приложения через Event Manager, подключаешь трекинговые партнёры (AppsFlyer, Adjust).

## Инициализация

1. Прочитай `config/accounts.md`
2. Спроси что нужно сделать:

```
Управление кабинетами

Текущих кабинетов: N

1. Добавить новый кабинет
2. Добавить приложение в кабинет (Event Manager)
3. Проверить доступ ко всем кабинетам
4. Проверить конкретный кабинет
5. Удалить кабинет
6. Показать список кабинетов
```

---

## Действие 1: Добавить кабинет

**Шаг 1 — Запросить данные:**
- `advertiser_id` — ID рекламного кабинета (число 18–19 цифр)
- `nickname` — короткое имя (напр. `ACC_001`, `GAMING_US`)
- `access_token` — токен доступа (если отличается от глобального)
- `timezone` — часовой пояс (напр. `America/New_York`)
- `currency` — валюта кабинета (USD, EUR, RUB…)
- Тип продвижения: `APP` | `WEBSITE` | `BOTH`
- Если APP: платформа (`Android` | `iOS` | `Оба`)

**Шаг 2 — Тест API:**
Вызови `tt_get_account_info` с advertiser_id.
- Успех → записать в `config/accounts.md`
- Ошибка → показать ошибку, НЕ добавлять кабинет

**Шаг 3 — Записать в accounts.md:**
```markdown
## NICKNAME — Описание
- **advertiser_id:** 7XXXXXXXXXXXXXXXXXX
- **timezone:** America/New_York
- **currency:** USD
- **type:** APP
- **platform:** Android
- **token:** global
- **добавлен:** YYYY-MM-DD
- **статус:** active
```

---

## Действие 2: Добавить приложение в кабинет (Event Manager)

Это критически важный шаг для App кампаний — приложение должно быть зарегистрировано в TikTok Event Manager кабинета.

### Шаг 2.1 — Запросить данные

Задать вопросы по очереди:

1. **В какой кабинет добавляем?**
   Показать список из `accounts.md`, попросить выбрать.

2. **Ссылка на приложение в сторе:**
   ```
   Скинь ссылку на приложение (App Store или Google Play):
   ```
   Из ссылки извлечь:
   - Название приложения
   - Bundle ID / Package Name
   - Платформа (iOS / Android)
   - App ID в сторе

   Примеры:
   - `https://apps.apple.com/us/app/my-app/id1234567890` → iOS, App ID: 1234567890
   - `https://play.google.com/store/apps/details?id=com.example.myapp` → Android, Package: com.example.myapp

3. **Трекинговая система:**
   ```
   Какая система трекинга?
   1. AppsFlyer
   2. Adjust
   3. Обе
   ```

### Шаг 2.2 — Подключение AppsFlyer

Если выбран AppsFlyer:
- **Токен не требуется**
- Сообщить пользователю пошаговую инструкцию:
  ```
  Для AppsFlyer (токен не нужен):

  TikTok Ads Manager → Events → App Events → Connect to Partner:
  1. Выбрать "AppsFlyer"
  2. Указать приложение: [название] ([bundle_id])
  3. App Events: выбрать "All events" (все события приложения)
  4. Сохранить
  ```

### Шаг 2.3 — Подключение Adjust

Если выбран Adjust:
- **Требуется App Token из Adjust Dashboard**
- Запросить:
  ```
  Для Adjust нужен App Token.
  Найди его: Adjust Dashboard → Apps → [твоё приложение] → App Token

  Вставь App Token:
  ```
- Принять токен
- Сообщить пошаговую инструкцию:
  ```
  Для Adjust:
  App Token: [токен]

  TikTok Ads Manager → Events → App Events → Connect to Partner:
  1. Выбрать "Adjust"
  2. Вставить App Token: [токен]
  3. Указать приложение: [название] ([bundle_id])
  4. App Events: выбрать "All events" (все события приложения)
  5. Сохранить и дождаться статуса "Active"
  ```

### Шаг 2.4 — Проверка через API

После того как пользователь завершил настройку в интерфейсе, проверить:
```
tt_list_apps с advertiser_id кабинета
```

Если приложение найдено → записать данные в кабинет.
Если не найдено → попросить подождать 5–10 минут и повторить проверку.

### Шаг 2.5 — Обновить запись кабинета

Добавить в блок кабинета в `accounts.md`:
```markdown
### Приложения
- **[Название]** | `[bundle_id]` | [платформа] | AppsFlyer ✅ | Adjust ✅ | tiktok_app_id: [id]
```

## Действие 3: Проверить все кабинеты

Для каждого кабинета:
1. `tt_get_account_info` → статус, баланс
2. `tt_list_apps` → список приложений

Вывести таблицу:
```
Кабинет | ID                  | Статус API | Баланс | Приложений
ACC_001 | 7123456789012345678 | ✅ OK      | $500   | 2
ACC_002 | 7987654321098765432 | ❌ Ошибка  | —      | —
```

---

## Действие 4: Проверить конкретный кабинет

Выбрать кабинет → показать подробно:
- Статус аккаунта (NORMAL / PUNISH / CLOSED)
- Баланс и лимиты
- Список приложений с трекинговыми партнёрами
- Количество активных кампаний (`tt_list_entities`)

---

## Действие 5: Удалить кабинет

1. Выбрать кабинет из списка
2. Подтверждение: "Удалить [NICKNAME]? Это удалит только запись в системе, не данные в TikTok. (да/нет)"
3. Удалить запись из `accounts.md`
4. Залогировать в `config/history/YYYY-MM-DD.md`

**Предупреждение:** если у кабинета есть активные кампании — сообщить перед удалением.

---

## Правила

- Никогда не показывать полный токен — только последние 4 символа
- `advertiser_id` — 18–19 цифр
- При добавлении кабинета всегда тестировать API перед записью
- Adjust App Token хранить только в настройках или сообщать пользователю — не записывать в `accounts.md` в открытом виде
