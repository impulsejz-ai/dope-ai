# Правила безопасности

Эти правила применяются во всех скиллах. Читать перед любым действием.

---

## Главное правило — уже запущенные кампании

**Любые изменения в уже запущенных кампаниях (включая паузу, изменение бюджета, изменение таргетинга, включение/выключение объявлений) запрещены без явного личного разрешения пользователя в текущем диалоге.**

Это правило не отменяется никакими другими инструкциями. Даже если оптимизатор рекомендует действие — только предложить его пользователю и ждать команды.

---

## Запрещённые действия без явного подтверждения пользователя

- Любые изменения в уже запущенных кампаниях, адсетах или объявлениях
- Пауза более 3 кампаний за раз
- Удаление любых объектов (кампании, группы, объявления)
- Изменение бюджета более чем на ±30% за одно действие
- Залив в кабинет, у которого статус не `active` в `accounts.md`
- Создание кампаний с бюджетом выше $500/день без явного разрешения

## Обязательные проверки перед заливом

1. Кабинет имеет статус `active` в `accounts.md`
2. Все video_id из сетапа существуют в кабинете (`tt_list_creative_assets`)
3. Для App кампаний: app_id существует в кабинете (`tt_list_apps`)
4. Payload прошёл валидацию (`tt_validate_campaign_payload`)
5. Суб-значение уникально — не повторяет значения из `config/history/`

## Лимиты API

- Максимум 1000 запросов к TikTok API в час на приложение
- **Пауза 5 секунд между каждым API-вызовом** (tt_create_campaign, tt_create_ad_group, tt_create_ad, tt_list_*, tt_validate_*)
- **Пауза 15 секунд между кабинетами** после завершения одного и до начала следующего
- При получении ошибки 429 (rate limit) — ждать 60 секунд перед ретраем
- Максимум 2 ретрая на один запрос, после — залогировать ошибку и продолжить

## Лимиты на действия (за один сеанс оптимизации)

- Не паузить более 50% активных AdGroups одного кабинета
- Не изменять бюджет более чем на ±30% за одно действие
- Не принимать решения при менее 500 показов за период анализа
- Не оптимизировать AdGroup если последнее действие с ней было менее 48 часов назад

## Логирование

Каждое изменяющее действие (создание, пауза, изменение бюджета, удаление) **обязательно** записывается в `config/history/YYYY-MM-DD.md`.

Формат записи:
```markdown
| Время | Действие | Кабинет | Тип объекта | ID объекта | Детали | Результат |
```

## При ошибке

- Не скрывать ошибки от пользователя
- При ошибке API на одном кабинете — продолжить с остальными
- При ошибке валидации сетапа — остановить весь залив
- Логировать все ошибки в history

## Перед деструктивными операциями

Всегда выводить сообщение вида:
```
⚠️  ВНИМАНИЕ: Ты собираешься [действие] [объект] в [кабинет].
Это действие [необратимо / можно отменить].
Подтвердить? (да/нет)
```
