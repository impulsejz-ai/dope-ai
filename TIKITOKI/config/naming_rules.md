# Правила нейминга

Нейминг — индивидуальный для каждого приложения/оффера. Агент всегда запрашивает шаблон перед заливом.

---

## Ключевой параметр: Sub-переменная

**Каждая кампания должна иметь уникальное значение суб-параметра.**

Суб-параметр — это трекинговый идентификатор кампании для партнёрской сети.

**Формат:** `{имя_параметра}={уникальное_значение}`

Примеры:
- `sub7=avfrites`
- `sub_id_3=rocketavtoza`
- `sub1=tikcamp001`

**Важно:**
- Имя параметра (`sub7`, `sub_id_3`) задаётся на уровне оффера/сетапа и не меняется
- Значение (`avfrites`, `rocketavtoza`) — **уникально для каждой кампании**
- Значение используется как в имени кампании, так и в трекинговой ссылке

### Где используется

1. **В имени кампании** — чтобы идентифицировать кампанию по суб-значению
2. **В трекинговой ссылке** — в URL кликтрекера (AppsFlyer / Adjust / direct)

Пример трекинговой ссылки с подставленным суб:
```
https://app.adjust.com/abc123?campaign=TT_{DATE}&sub7=avfrites&...
```

## Шаблон нейминга

При запуске `/mass-uploader` агент запрашивает у пользователя:

1. **Шаблон имени кампании** — пример: `TT_{APP}_{DATE}_{SUB}`
2. **Шаблон имени AdGroup** — пример: `{CAMPAIGN}_{GEO}_{PLATFORM}_AG{INDEX}`
3. **Шаблон имени Ad** — пример: `{ADGROUP}_V{INDEX}`
4. **Имя суб-параметра** — пример: `sub7` или `sub_id_3`
5. **Суб-значение для каждой кампании** — уникальное, задаётся вручную или генерируется

### Автогенерация суб-значения

Если пользователь хочет автогенерацию — агент создаёт уникальную строку:
- Формат: `tt{YYYYMMDD}{4_случайных_буквы}` — пример: `tt20250501xkrp`
- Или пользователь предоставляет список суб-значений (по одному на кабинет)

---

## Переменные для подстановки в шаблонах

| Переменная | Значение |
|-----------|---------|
| `{DATE}` | YYYYMMDD — дата создания |
| `{APP}` | Короткое имя приложения из сетапа |
| `{GEO}` | Код гео из сетапа (US, EU, GLOBAL…) |
| `{PLATFORM}` | AND / IOS / ALL |
| `{INDEX}` | Порядковый номер 01, 02… |
| `{ACCOUNT}` | Никнейм кабинета |
| `{SUB}` | Уникальное суб-значение кампании |
| `{CREATIVE}` | Метка или порядковый номер видео |

---

## Хранение нейминга в сетапе

Нейминг записывается в файл сетапа в секцию `naming`:

```yaml
naming:
  campaign: "TT_{APP}_{DATE}_{SUB}"
  adgroup: "{CAMPAIGN}_{GEO}_{PLATFORM}_AG{INDEX}"
  ad: "{ADGROUP}_V{INDEX}"
  sub_param: "sub7"          # имя параметра (не меняется в рамках оффера)
  sub_values: []             # заполняется при заливе (по одному на кабинет)
                             # или оставить пустым для автогенерации
```

---

## Ограничения TikTok

- Кампания: не более 512 символов
- AdGroup / Ad: не более 255 символов
- Допустимые символы: латиница, цифры, `_`, `-`, пробел (ASCII)
